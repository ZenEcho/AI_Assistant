# 系统输入增强测试用例

这份文档用于验证“外部应用输入框翻译”这条链路是否可靠，重点覆盖：

- 触发
- 捕获
- 翻译
- 回写
- 失败回退
- 快捷键动作
- 目标语言悬浮窗

## 1. 适用范围

- 主验证平台：`Windows`
- 当前主能力：原生捕获、选中翻译、剪贴板翻译、自动回写、结果窗回退
- 当前非强验范围：
  - macOS / Linux
  - 富文本编辑器中的高稳定原生替换
  - `manual-hotkey` 触发模式的完整闭环

## 2. 功能矩阵

| 能力 | 当前状态 | 备注 |
| --- | --- | --- |
| 双 `Alt` 触发系统输入会话 | 已实现 | 推荐主回归路径 |
| 双空格触发系统输入会话 | 已实现 | 更容易受输入法与编辑器影响 |
| `manual-hotkey` 触发模式 | 未完整闭环 | UI 已暴露，按 Known Gap 处理 |
| 选中翻译快捷键 | 已实现 | 可选自动覆盖原文 |
| 剪贴板翻译快捷键 | 已实现 | 生成译文后等待粘贴 |
| 粘贴上次译文快捷键 | 已实现 | 复用最近一次快捷键译文 |
| 开关系统输入增强快捷键 | 已实现 | 有系统通知反馈 |
| `Ctrl+~` 目标语言悬浮窗 | 已实现 | 支持循环切换目标语言 |
| 自动回写策略 | 已实现 | 原生替换 -> 模拟输入 -> 剪贴板回退 |

## 3. 测试前置

### 应用前置

- 已启动桌面应用
- 至少有一个可用模型
- 设置页中“系统输入增强”已打开
- 建议开启调试日志，便于观察最近错误与原生状态

### 推荐测试应用

- `Notepad`：纯文本输入框，最适合做主回归
- `Chrome / Edge textarea`：浏览器文本框
- `微信 / 钉钉 / 飞书`：聊天输入框
- `VS Code`：用于验证“排除代码编辑器”

### 推荐基线配置

- 触发方式：`double-alt`
- 双击间隔：`280ms`
- 文本获取策略：`before-caret-first`
- 文本回填策略：`auto`
- 仅处理选中文本：`off`
- 自动替换原文：`on`
- 启用剪贴板回退：`on`
- 显示轻提示：`on`
- 仅英文触发：`off`
- 排除代码编辑器：`on`
- 目标语言：`English`

## 4. 自动化 Smoke

如果是回归原生主链路，建议优先跑脚本：

```bash
pnpm build:desktop:smoke
pnpm smoke:system-input:notepad:smokeapp
```

如果只是本地快速验证：

```bash
pnpm smoke:system-input:notepad
```

Smoke 脚本主要用于判断“主链路还活着”，不能替代全部手工回归。

## 5. 手工用例

### 5.1 启停与基础触发

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-001` | 关闭后不触发 | 关闭系统输入增强，在 Notepad 输入 `hello world`，按两次 `Alt` | 不触发翻译，不回写，不弹结果窗 |
| `SI-002` | 开启后可触发 | 开启系统输入增强，在 Notepad 输入 `hello world`，按两次 `Alt` | 触发翻译；成功则直接替换，失败则回退结果窗 |
| `SI-003` | 双击间隔生效 | 双击间隔设为 `280ms`，两次 `Alt` 间隔 1 秒 | 不应触发 |

### 5.2 触发模式

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-010` | 双 `Alt` | 配置 `double-alt`，在 Notepad 输入文本后按两次 `Alt` | 应稳定触发 |
| `SI-011` | 双空格 | 配置 `double-space`，输入文本后快速按两次空格 | 应触发；触发空格不应残留为脏字符 |
| `SI-012` | 手动热键 | 配置 `manual-hotkey` 并尝试触发 | 当前按 Known Gap 记录；无响应不算普通回归缺陷 |

### 5.3 捕获策略

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-020` | 选中文本优先 | 选择 `world` 后触发 | 优先只翻译并替换选中部分 |
| `SI-021` | 光标前优先 | 行尾不选中，直接触发 | 优先捕获光标前文本 |
| `SI-022` | 整段输入优先 | 在多行输入框中不选中，直接触发 | 尽量捕获整个输入内容 |
| `SI-023` | 仅处理选中文本 | 开启后不选中文本直接触发 | 不应退化为整段捕获，应给出失败提示或回退结果窗 |

### 5.4 回写策略

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-030` | `auto` | 使用默认 `auto` 回写 | 按 `native -> simulate -> clipboard` 尝试，成功则不抢焦点 |
| `SI-031` | `native-replace` | 指定原生替换 | 成功则直接替换；失败则回退结果窗 |
| `SI-032` | `simulate-input` | 指定模拟输入 | 删除原文后重新输入译文；失败则回退 |
| `SI-033` | `clipboard-paste` | 指定剪贴板粘贴 | 应替换成功，并在结束后恢复用户原剪贴板内容 |
| `SI-034` | `popup-only` | 指定仅结果窗 | 不应尝试修改外部输入框，只显示结果窗 |

### 5.5 快捷键动作

默认快捷键来自系统输入增强配置页，基线值通常为：

- `Ctrl+1`：翻译选中内容
- `Ctrl+2`：翻译剪贴板
- `Ctrl+3`：粘贴上次译文
- `Ctrl+4`：开关系统输入增强

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-040` | 选中翻译快捷键 | 选中外部文本后按 `Ctrl+1` | 生成译文；如开启“选中翻译后自动覆盖”，应直接替换 |
| `SI-041` | 剪贴板翻译快捷键 | 复制文本后按 `Ctrl+2` | 应生成译文并提示可粘贴 |
| `SI-042` | 粘贴上次译文 | 先执行 `Ctrl+1` 或 `Ctrl+2`，再按 `Ctrl+3` | 应将最近译文粘贴到当前外部输入焦点 |
| `SI-043` | 开关快捷键 | 按 `Ctrl+4` | 应切换启用状态并给出系统通知 |
| `SI-044` | 重复内容复用 | 对同一段选中文本连续触发两次快捷翻译 | 第二次应优先复用上次译文，而不是再次发请求 |

### 5.6 目标语言悬浮窗

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-050` | 呼出悬浮窗 | 按 `Ctrl+~` | 显示当前系统输入目标语言 |
| `SI-051` | 连续切换 | 悬浮窗显示期间再次按 `Ctrl+~` | 切换到下一个目标语言 |
| `SI-052` | 自动目标说明 | 目标语言为 `auto` 时呼出 | 应显示“优先翻译到系统语言；若原文就是系统语言，则翻译为英语”的说明 |
| `SI-053` | 自动关闭 | 鼠标移出悬浮窗区域并停留约 2 秒 | 悬浮窗自动隐藏 |

### 5.7 过滤与安全边界

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-060` | 仅英文触发 | 开启后在 Notepad 输入中文触发 | 不应继续翻译，应给出明确提示 |
| `SI-061` | 排除代码编辑器 | 开启后在 VS Code 编辑器中触发 | 不应触发系统输入增强 |
| `SI-062` | 黑名单 | 将 `notepad.exe` 加入黑名单后触发 | 不应触发 |
| `SI-063` | 白名单 | 白名单只保留 `WINWORD.EXE` 后在 Notepad 触发 | 不应触发 |
| `SI-064` | 应用自排除 | 在本应用主窗、设置页、结果窗内按触发键 | 不应误走系统输入增强链路 |

### 5.8 模型与失败回退

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `SI-070` | 当前模型优先 | 主翻译页选择一个非默认模型后触发系统输入 | 应优先使用当前选中模型 |
| `SI-071` | 默认模型兜底 | 当前选中模型故意配错，默认模型可用 | 失败后应回退到默认模型 |
| `SI-072` | 无模型 | 删除或禁用所有模型后触发 | 应给出“请先添加并启用至少一个模型”之类提示 |
| `SI-073` | 回写失败回退 | 在不支持原生替换的输入控件触发 | 应回退到结果窗，而不是静默失败 |

## 7. 已知边界

- `manual-hotkey` 当前仍按“已暴露 UI、但尚未完整闭环”处理。
- 双空格比双 `Alt` 更容易受到输入法、富文本编辑器和输入习惯影响。
- 在复杂富文本编辑器中，原生替换与模拟输入失败并不一定代表回归，只要能明确回退到结果窗即可。
- 当前“轻提示”本质上仍有结果窗参与，不等价于真正系统级 Toast。

## 8. 建议记录格式

```text
SI-010 | Notepad | double-alt / before-caret-first / auto | 触发成功并完成回写 | Pass
SI-042 | Chrome textarea | shortcut Ctrl+3 | 上次译文成功粘贴 | Pass
SI-061 | VS Code | excludeCodeEditors=on | 未触发系统输入增强 | Pass
SI-012 | Notepad | manual-hotkey | 无响应，按 Known Gap 记录 | Known Gap
```

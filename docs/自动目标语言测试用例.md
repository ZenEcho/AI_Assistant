# 自动目标语言测试用例

这份文档用于验证“目标语言 = `auto`”时的真实行为。这里的重点不是“自动检测源语言”本身，而是“检测之后如何决定目标语言”。

## 1. 当前实现规则

自动目标语言的实现位于：

- [`src/services/ai/translationRequestResolver.ts`](../src/services/ai/translationRequestResolver.ts)
- [`src/services/ai/languageDetectionService.ts`](../src/services/ai/languageDetectionService.ts)
- [`src/services/ai/translationResolutionFormatter.ts`](../src/services/ai/translationResolutionFormatter.ts)

### 规则本体

当 `targetLanguage !== auto` 时：

- 不走自动目标语言逻辑
- 直接按手动目标语言翻译

当 `targetLanguage === auto` 时：

- 如果源语言与系统语言一致 -> 目标语言解析为 `English`
- 如果源语言与系统语言不同 -> 目标语言解析为系统语言
- 如果检测不可靠：
  - 短文本 -> 回退到系统语言
  - 混合文本 -> 回退到系统语言
  - 低置信度 -> 回退到系统语言

### 检测顺序

1. 若用户手动指定了源语言，则直接使用手动语言
2. 先尝试启发式检测
3. 启发式不可靠时，再调用模型做语言检测
4. 模型检测失败时，回退启发式或 `und`

## 2. 预期展示形式

除了译文本身，这个功能还会影响：

- 结果页标签：显示 `自动 -> 实际目标语言`
- 结果页说明文案：显示自动决策原因
- 历史记录：保留原始请求和解析结果
- 系统输入通知 / 结果窗：提示自动决策结果

## 3. 测试前置

### 应用前置

- 已启动桌面应用
- 至少存在一个可用模型
- 主翻译页当前选中模型可正常请求
- 目标语言已设置为 `auto`

### 推荐系统环境

至少准备两种系统语言环境：

- 中文系统，例如 `zh-CN` 或 `zh-TW`
- 英文系统，例如 `en-US`

### 推荐测试文本

- 中文：`这是一次中文测试`
- 英文：`hello world`
- 日文：`これは日本語のテストです`
- 韩文：`이것은 한국어 테스트입니다`
- 短英文：`ok`
- 短中文：`嗯`
- 混合文本：`hello 你好 world`

## 4. 主翻译页用例

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `AT-001` | 中文系统 + 中文输入 | 系统语言为中文，目标语言设为 `auto`，输入中文翻译 | 结果应翻译为 `English`，标签显示 `自动 -> English` |
| `AT-002` | 中文系统 + 英文输入 | 中文系统下输入 `hello world` | 结果应翻译为中文 |
| `AT-003` | 中文系统 + 日文输入 | 中文系统下输入日文 | 结果应翻译为中文 |
| `AT-004` | 英文系统 + 英文输入 | 英文系统下输入英文 | 结果仍翻译为 `English`，并给出“与系统语言一致，因此翻译为 English”的说明 |
| `AT-005` | 英文系统 + 日文输入 | 英文系统下输入日文 | 结果应翻译为 `English` |
| `AT-006` | 短文本回退 | 输入 `ok` | 若检测不稳定，应回退为系统语言，并显示短文本回退说明 |
| `AT-007` | 非拉丁短文本 | 中文系统下输入 `嗯` | 尽量识别为中文，从而翻译为 `English` |
| `AT-008` | 混合文本回退 | 日文系统下输入 `hello 你好 world` | 若无法稳定识别主语言，应回退翻译为日文，并显示 mixed fallback 文案 |
| `AT-009` | 手动目标语言覆盖自动逻辑 | 目标语言改成 `Japanese` 后翻译中文 | 不应再出现自动目标说明，直接翻译为日文 |
| `AT-010` | 手动源语言优先 | 手动源语言设为 `Chinese (Traditional)`，目标语言为 `auto` | 应保留该手动源语言语义，不应被系统语言反向改写 |

## 5. 图片翻译用例

自动目标语言不仅适用于纯文本，也适用于图片翻译。

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `AT-020` | 图片中文 -> 英文 | 中文系统下粘贴一张中文截图，目标语言为 `auto` | 应翻译为 `English` |
| `AT-021` | 图片英文 -> 中文 | 中文系统下粘贴一张英文截图，目标语言为 `auto` | 应翻译为中文 |
| `AT-022` | 图片 + 额外文本 | 上传图片并补充一段英文说明文本 | 自动目标语言仍按整体源内容判断，结果标签与说明保持一致 |

## 6. 结果页与历史记录用例

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `AT-030` | 结果页标签 | 完成任意一次 `auto` 翻译 | 结果页显示 `自动 -> 实际目标语言` |
| `AT-031` | 结果页说明文案 | 完成一次 `source-equals-system` 或 fallback 场景 | 应展示对应说明，而不是只显示结果 |
| `AT-032` | 历史记录保留解析结果 | 完成一次 `auto` 翻译后查看历史 | 历史项应保留 `resolution` 相关信息 |
| `AT-033` | 历史重放重新解析 | 记录一条 `auto` 翻译历史后点击重放 | 应基于原始 `auto` 请求重新解析，而不是把旧的 resolved target 写死复用 |

## 7. 系统输入增强用例

系统输入增强同样会走自动目标语言逻辑。

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| `AT-040` | 系统输入目标语言 = `auto` | 设置页把系统输入目标语言改成 `auto` | 保存成功，悬浮窗与通知都能识别 `auto` |
| `AT-041` | `Ctrl+~` 悬浮窗说明 | 目标语言为 `auto` 时呼出悬浮窗 | 显示“优先翻译到系统语言；若原文就是系统语言，则翻译为英语” |
| `AT-042` | 中文系统下快捷翻译英文 | 选中英文文本并触发系统输入快捷翻译 | 应提示“检测到 English，已自动翻译为中文”之类说明 |
| `AT-043` | 中文系统下快捷翻译中文 | 选中中文文本并触发系统输入快捷翻译 | 应提示已自动翻译为 `English` |
| `AT-044` | 通知不覆盖动作提示 | 使用系统输入快捷翻译 | 通知既要说明自动决策结果，也要保留后续操作提示（如可按快捷键粘贴） |

## 8. 结果原因覆盖

建议至少手工覆盖下面几种 `reason`：

| reason | 代表场景 |
| --- | --- |
| `source-equals-system` | 中文系统输入中文；英文系统输入英文 |
| `source-differs-from-system` | 中文系统输入英文 / 日文；英文系统输入中文 / 日文 |
| `short-text-fallback` | `ok` 等极短文本 |
| `mixed-language-fallback` | `hello 你好 world` |
| `low-confidence-fallback` | 文本较短但又不是典型短词，或检测结果波动较大 |
| `manual-target` | 目标语言不是 `auto` |
